# Snapfield 설치 가이드

Windows PC(EXE) · Android(APK) · 리눅스(수신 데몬) — v0.16.7 기준

**Snapfield**는 여러 PC와 안드로이드·리눅스 기기의 화면을 하나의 물리 평면에 놓고, 마우스·키보드·클립보드를 화면 경계 너머로 공유하는 도구다. 조작하는 Windows PC 1대(**조작 기기**)와, 조작을 받는 PC/폰/리눅스(**수신 기기**) 여러 대로 구성된다.

> **다운로드**: 모든 파일은 [GitHub 릴리스](https://github.com/Q07K/Snapfield/releases)에서.
> 최신 릴리스의 `Snapfield-vX.Y.Z.exe`(PC), `Snapfield-vX.Y.Z.apk`(안드로이드),
> `Snapfield-Receiver-vX.Y.Z-linux-{x64,arm64}`(리눅스)를 받는다.
> 모두 설치 과정이 없는 단일 파일이다 (.NET 등 별도 설치 불필요).

## 1. Windows PC 설치 (EXE)

1. 릴리스 페이지에서 `Snapfield-vX.Y.Z.exe`를 다운로드해 원하는 폴더에 둔다.
2. 실행하면 **SmartScreen 경고**가 뜰 수 있다 → **추가 정보 → 실행**. (코드 서명이 없는 exe라 표시되며, 기기당 한 번)
3. 이어서 **UAC 관리자 승인 창**이 뜬다 → **예**. (v0.15부터 앱이 관리자 권한으로 실행된다 — 작업 관리자·관리자 명령창 같은 관리자 권한 창 위에서도 원격 조작이 끊기지 않기 위해 필요. 실행할 때마다 한 번 뜨며, 아래 "로그인 시 자동 실행"을 켜면 부팅 후에는 UAC 없이 시작된다.)
4. 첫 실행 후 역할을 선택한다 — 왼쪽 **조작 기기**(이 PC의 마우스·키보드로 다른 기기 조작) 또는 오른쪽 **수신 기기**(조작 받기, 선택 즉시 대기 시작).
5. 방화벽 규칙(같은 네트워크 기기 자동 발견에 필요)은 자동으로 등록된다 — 별도 창 없음.

> ✅ **업데이트**: 새 버전 exe를 받아 기존 파일을 교체하면 끝. 방화벽 규칙도 새 exe 경로로 자동 갱신된다.
> ⚠ **v0.13.9 이상으로 올릴 때**: 통신 프로토콜이 바뀌어 구버전과 연결이 유지되지 않는다 — **조작·수신 기기를 함께 업데이트**할 것.

창을 닫아도 트레이에서 연결이 유지된다. 완전 종료는 **트레이 아이콘 우클릭 → 종료**.
설정 탭의 **로그인 시 자동 실행**은 작업 스케줄러에 등록되어, 부팅 후 UAC 창 없이 관리자 권한으로 시작된다.

## 2. Android 설치 (APK)

안드로이드 폰/태블릿이 평면 위의 모니터가 된다 — PC 커서가 화면 경계를 넘어 들어오고, 탭·드래그·스크롤·키보드·클립보드가 동작한다.

### 2-1. 설치

1. 폰 브라우저로 릴리스 페이지에서 `Snapfield-vX.Y.Z.apk`를 다운로드한다.
2. 파일을 열어 설치를 시도하면 **"출처를 알 수 없는 앱"** 안내가 나온다 → 해당 브라우저(또는 파일 관리자)의 **출처 허용**을 켠다.
3. **Google Play 프로텍트가 설치를 차단**하면: Play 스토어 → 프로필 → Play 프로텍트 → 설정(톱니) → **"Play 프로텍트로 앱 검사" 끄기** → APK 다시 설치 → **검사 다시 켜기**.

### 2-2. 앱 설정 (설치 후 1회)

앱을 열면 **설정 체크리스트**가 순서대로 안내한다:

1. **수신 시작** — 탭 한 번으로 연결 대기가 시작된다.
2. **접근성 권한 켜기** (커서·탭에 필요) — 처음에는 **"제한된 설정"** 으로 막힌다:
   설정 → 애플리케이션 → Snapfield → **⋮(더보기) → 제한된 설정 허용** → 그다음 설정 → 접근성에서 Snapfield를 켠다.
3. **Snapfield 키보드 켜기** — PC 키보드로 입력하려면 필요하다 (한글 두벌식 지원).
4. 텍스트를 입력할 때는 **Snapfield 키보드로 전환** — 화면 키보드가 사라지고 PC에서만 입력된다. (폰을 직접 쓸 땐 원래 키보드로 되돌리면 됨)

> ⚠ **업데이트 후 접근성 한 번 껐다 켜기**: APK를 업데이트하면 설정 → 접근성에서 Snapfield를 **껐다가 다시 켜야** 화면 캡처(PrtSc) 기능이 동작한다. 접근성 능력이 서비스를 켤 때 읽히기 때문.

> ✅ **업데이트**: 새 APK를 받아 그대로 설치하면 **삭제 없이 덮어쓰기**된다 (v0.13.3 이후 정식 서명). 그 이전 버전에서 올릴 때만 기존 앱을 삭제 후 설치 (연결 코드가 재생성됨).

## 3. 리눅스 설치 (수신 데몬)

우분투 데스크톱(Wayland/X11)이 평면 위의 모니터가 된다. 입력 주입이 커널 레벨(`/dev/uinput` 가상 마우스·키보드)이라 진짜 시스템 커서가 움직인다. GUI 앱이 아니라 **터미널에서 실행하는 콘솔 데몬**이다 — 파일 더블클릭으로는 실행되지 않는다.

### 3-1. 어느 파일을 받나

터미널에서 `uname -m`:

- `x86_64` (일반 인텔/AMD PC) → `Snapfield-Receiver-*-linux-x64`
- `aarch64` (ASUS Ascent GX10, NVIDIA DGX Spark, 라즈베리파이 5 등 ARM 기기) → `Snapfield-Receiver-*-linux-arm64`

아키텍처가 다른 파일은 `Exec format error`로 실행되지 않는다.

### 3-2. 실행

데스크톱 세션 안에서 터미널을 열고 (SSH가 아니라 그 기기의 화면에서):

```bash
cd ~/Downloads
chmod +x Snapfield-Receiver-*-linux-*    # 최초 1회 (브라우저 다운로드는 실행 권한이 없음)
./Snapfield-Receiver-*-linux-*
```

최초 실행 시 부족한 것들을 **자동 설정으로 제안**한다 — 각각 **Y** + sudo 암호 한 번이면 끝:

1. **uinput 권한** — udev 규칙 설치 (재로그인 없이 즉시 적용)
2. **클립보드 패키지** — 세션에 맞춰 Wayland는 `wl-clipboard`, X11은 `xclip` (없으면 클립보드 동기화만 꺼진 채 동작)

거절하면 수동 설정 명령이 출력된다. 실행되면 **연결 코드 6자리**와 이 기기의 IP가 표시된다 (코드는 `~/.config/snapfield/receiver.json`에 유지).

> ⚠ **터미널 창을 닫지 말 것** — 데몬이 그 안에서 돈다. 중지는 `Ctrl+C`.
> 화면 크기가 이상하게 잡히면 `--size 2560x1440`처럼 수동 지정 (`--help` 참고).

### 3-3. 동작 범위

- 커서·클릭·스크롤·키보드 전체. **한/영 키**도 리눅스 쪽 ibus 한글 입력기로 그대로 토글된다
- 클립보드 텍스트·이미지 **양방향** (v0.16.6+에서 X11 세션 지원 — 전송 시 터미널에 `클립보드 전송` 로그가 찍힌다)
- 업데이트: 새 버전 파일을 받아 `chmod +x` 후 기존 파일 대신 실행하면 끝 (연결 코드는 유지됨)

## 4. 기기 연결하기

1. **수신 기기**(PC·폰·리눅스)에서 수신을 시작하면 화면(리눅스는 터미널)에 **IP와 연결 코드 6자리**가 표시된다.
2. **조작 기기** PC의 연결 탭 → **기기 추가** 목록에 수신 기기가 **● 온라인**으로 자동 표시된다 → 탭. (안 보이면 IP로 직접 연결)
3. 수신 기기 화면의 **초록 숫자 6자리**를 입력하면 자동으로 연결된다. 한 번 연결한 기기는 다음부터 **원탭 재연결**.
4. **모니터 배치 탭**에서 기기들을 실제 책상 배치대로 드래그 — 놓는 즉시 저장되고 연결에 실시간 반영된다. 드래그 중에는 커서가 통과할 수 있는 경계가 초록으로 표시된다. 마우스 휠로 확대 가능.
5. 커서를 화면 경계로 밀면 상대 기기로 넘어간다. 끝!

## 5. 단축키 · 키 매핑

| PC 입력 | 동작 |
|---|---|
| 커서를 화면 끝으로 | 인접한 기기로 넘어감 |
| `Ctrl+Alt+←/→` | **기기 전환 스트립** — Ctrl+Alt를 누른 채 화살표로 기기 선택, 떼면 이동, `Esc`는 취소. 커서가 도착한 자리에 펄스가 표시됨 |
| `Ctrl` 3번 연타 | 원격 조작 중 즉시 내 PC로 복귀 (비상 탈출) |

**안드로이드를 조작할 때** 추가로:

| PC 입력 | 폰 동작 |
|---|---|
| 마우스 우클릭 | 뒤로 가기 |
| 마우스 휠클릭 | 최근 앱 |
| `Esc` | 뒤로 가기 |
| `Win` | 홈 화면 |
| `Alt+Tab` | 최근 앱 전환 |
| `PrtSc` | 폰 화면 캡처 → **PC 클립보드로** (PC에서 바로 붙여넣기) |
| 한/영 키 | 한글 ↔ 영문 입력 전환 (두벌식) |
| 볼륨/음소거/미디어 키 | 폰 볼륨·재생 제어 |

> **클립보드**: 텍스트·이미지가 양방향으로 공유된다 (PC↔PC는 파일도). 폰→PC 방향은 안드로이드 정책상 **Snapfield 키보드 사용 중이거나 Snapfield 앱이 화면에 있을 때** 복사분이 전송된다.

## 6. 문제 해결

| 증상 | 해결 |
|---|---|
| 기기 추가에 안 뜸 | 수신 기기가 대기 중인지 확인 → 조작 기기 PC에서 연결 탭에 다시 진입(방화벽 UAC가 뜨면 허용) → 같은 네트워크(Wi-Fi)인지 확인. 그래도 안 되면 IP로 직접 연결 |
| APK 설치가 차단됨 | 2-1의 Play 프로텍트 절차 참고 (검사 잠시 끄기 → 설치 → 켜기) |
| 접근성을 못 켬 | "제한된 설정" 잠금 상태 — 2-2의 2단계 절차로 해제 |
| PrtSc 캡처가 안 됨 | 접근성에서 Snapfield를 껐다 다시 켜기 (업데이트 후 필수) |
| 연결 코드가 안 맞음 | 수신 기기 화면의 **현재** 코드를 다시 확인 (재설치·재발급 시 코드가 바뀜). 저장된 최근 연결이 실패하면 새 코드로 다시 연결 |
| 폰에서 커서가 안 보임/안 눌림 | 접근성 권한 확인 → 폰 화면이 켜져 있는지 확인 (앱의 "조작 중 화면 유지" 옵션 권장) |
| 키보드 입력이 안 됨 | 입력창에서 키보드를 **Snapfield 키보드**로 전환했는지 확인 |
| 커서가 사라져 못 찾음 | `Ctrl` 3연타로 조작 PC 복귀. 그래도 안 보이면 Snapfield를 다시 실행 (실행 시 커서 자동 복원) |
| 연결 직후 입력하자마자 끊김 | 두 기기의 버전 불일치 (v0.13.9에서 프로토콜 변경) — **조작·수신 기기 모두 최신 버전으로 업데이트** |
| 작업 관리자 등 관리자 창 위에서 조작이 안 됨 | v0.15 이상인지 확인 — 관리자 실행이 필요한 기능. UAC 창에서 "예"를 눌렀는지 확인 |
| 리눅스: `Exec format error` | 아키텍처가 다른 파일 — `uname -m` 확인 후 `aarch64`면 `-linux-arm64`, `x86_64`면 `-linux-x64`를 받기 (3-1 참고) |
| 리눅스: `/dev/uinput` 권한 오류 | 최초 실행의 자동 설정에서 Y를 눌렀는지 확인 (v0.16.4+ 필요 — 이하 버전은 자동 설정에 버그). 거절했다면 출력된 수동 명령 실행 |
| 리눅스: 클립보드가 안 넘어감 | 세션 확인(`echo $XDG_SESSION_TYPE`) 후 Wayland는 `sudo apt install wl-clipboard`, X11은 `sudo apt install xclip` → 데몬 재시작. X11 지원은 v0.16.6+, 리눅스→PC 방향은 v0.16.7+ 권장. 마우스 드래그 선택만으로는 동기화 안 됨 — **Ctrl+C**(터미널은 Ctrl+Shift+C)로 복사해야 함 |
