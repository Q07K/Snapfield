# 릴리스 히스토리

버전별 주요 변경 사항 — 최신이 위. 다운로드는 [GitHub 릴리스](https://github.com/Q07K/Snapfield/releases)에서.

> 표기: **exe** = Windows 앱, **apk** = 안드로이드 수신 앱, **linux** = 리눅스 수신 데몬.
> 버전이 붙는 자산만 바뀐 릴리스는 다른 자산을 함께 업데이트할 필요가 없다 (예: linux만 바뀐 v0.16.4~7은 PC exe를 올리지 않아도 됨).

## v0.16 — 리눅스 수신 기기

우분투 데스크톱(Wayland/X11)을 평면 위 모니터로 만드는 헤드리스 수신 데몬 `Snapfield-Receiver-*-linux-{x64,arm64}` 추가.

- **v0.16.7** (linux): X11 클립보드 변경 감지를 형식 목록(TARGETS) 조회 없이 내용 직접 확인 방식으로 — 일부 환경에서 리눅스→PC 방향이 조용히 막히던 문제 해결. 전송 시 터미널에 `클립보드 전송 (N자)` 로그 표시
- **v0.16.6** (linux): **X11 세션 지원** — 클립보드 백엔드를 세션에 따라 선택 (Wayland는 wl-clipboard 감시, X11은 xclip 1초 폴링). NVIDIA 드라이버 데스크톱(GNOME on Xorg 등)에서 클립보드 동기화 동작
- **v0.16.5** (linux): 최초 실행 시 클립보드 동기화 패키지(wl-clipboard/xclip)가 없으면 **자동 설치 제안** (Y + sudo 한 번)
- **v0.16.4** (linux): 최초 실행 uinput 자동 설정 스크립트가 CRLF 빌드에서 `set: Illegal option -`으로 실패하던 버그 수정. 수동 설정 안내에 `modprobe uinput` 단계 추가
- **v0.16.3** (linux): **ARM64(aarch64) 빌드 추가** — ASUS Ascent GX10, NVIDIA DGX Spark, 라즈베리파이 5 등에서 실행 가능 (`uname -m`이 `aarch64`면 `*-linux-arm64`를 받을 것). ⚠ v0.16.2 태그는 잘못된 커밋에 푸시되어 건너뜀
- **v0.16.1** (linux): 최초 실행 시 `/dev/uinput` 권한을 **스스로 설정 제안** — udev `uaccess` 규칙이라 재로그인 없이 즉시 적용
- **v0.16.0** (linux): **리눅스 수신 데몬 첫 릴리스** — 입력 주입이 커널 레벨(`/dev/uinput` 가상 마우스·키보드)이라 Wayland에서도 컴포지터와 무관하게 동작. 연결 코드·암호화·발견 비컨은 데스크톱과 동일 프로토콜, 클립보드 텍스트·이미지 양방향, xrandr 화면 감지(`--size`로 수동 지정 가능), 한/영 키가 리눅스 ibus 입력기로 그대로 토글

## v0.15 — 관리자 실행 + 대각 점프 수정

- **v0.15.2** (apk): `Enter`가 입력 필드의 IME 액션(보내기 등)을 수행, `Shift+Enter` = 줄바꿈
- **v0.15.1** (apk): 안드로이드 수신 기기가 v0.13.9+ 바이너리 fast-path를 다시 이해하도록 수정 (구버전 apk는 첫 입력에서 끊김)
- **v0.15.0** (exe):
  - **대각 점프 라우팅 수정** — 경계 너머 최근접 모니터 선택에 측면 거리 상한이 없어 순수 대각선 방향 기기로 점프하던 문제 수정 (배치 캔버스의 통과 표시도 동일 규칙)
  - **관리자 권한으로 실행** (UIPI 대응) — 관리자 창(작업 관리자 등)에 포커스가 있으면 후킹·주입이 막혀 원격 조작이 죽던 문제 해결. 실행 시 UAC 1회. 자동 실행이 레지스트리 Run 키 → **작업 스케줄러 로그온 태스크**(UAC 없이 관리자로 시작)로 이전, 기존 등록은 첫 실행 시 자동 마이그레이션

## v0.14 — 기기 전환 UX

- `Ctrl+Alt+←→`가 Alt+Tab식 **전환 스트립**을 띄움 — 로컬·원격 화면 양쪽 표시, 누른 채 화살표로 선택 → 떼면 이동 → Esc 취소
- 이동 후 커서 도착 지점에 **랜딩 펄스**
- 원격 조작 중에도 전환 핫키가 동작하도록 수정(0.14.1), 전환 시 modifier 키가 눌린 채 남던 문제 수정

## v0.13 — 안드로이드 수신 기기 + 엔진 안정화

- **v0.13.0~0.13.6**: 안드로이드 수신 앱 — 모노레포 `android/`(코틀린, 의존성 0) + 릴리스에 APK 자동 첨부
  - 데스크톱과 동일한 와이어 프로토콜(ECDH→AES-256-GCM, 발견 비컨)을 코틀린으로 이식 — PC "기기 추가"에 자동 표시
  - 접근성 서비스로 커서 오버레이 + 탭/드래그/스크롤 재현, 화면 없는 IME로 키보드(**두벌식 한글 조합** 포함)
  - 클립보드 텍스트·이미지 양방향, PrtSc→폰 화면 캡처를 PC 클립보드로, 조작 중 화면 유지
  - 기기 종류(스마트폰/태블릿) 자동 판별 → 배치 캔버스 실루엣, 설정 체크리스트 온보딩, 상태 히어로
  - CI 키스토어 서명 — 덮어쓰기 업데이트 가능
  - 후속 패치: 커서-탭 좌표 정렬, 안드로이드식 키매핑(우클릭=뒤로 등), 클립보드 이미지 에코로 붙여넣기가 죽던 문제 수정, 원격→원격 경계의 틈 통과, **조용한 캔버스**(경계 표시는 드래그 중에만 + 작은 카드는 이름만 크게)
- **v0.13.7**: 캡처 중 먹통 방지 — 수신 기기가 멈춰도(TCP 블랙홀) 후킹 스레드 데드락 없이 복구. 연결 해제 이벤트 전용 스레드 분리, OS가 몰래 제거한 훅을 감시견이 재설치, 전송 타임아웃 5초
- **v0.13.8**: 엔진 웨지 방지 — SendInput 재진입 소용돌이 차단, 비상 복귀(`Ctrl` 3연타)를 락 없이 즉시 실행, **단일 인스턴스 뮤텍스**
- **v0.13.9**: **바이너리 fast-path** — 커서/버튼/휠/키 메시지를 JSON 대신 고정 바이너리로(커서 이동 ~45B → 9B). ⚠ 구버전 수신 기기와는 첫 입력에서 연결이 끊기므로 **양쪽을 함께 업데이트**

## v0.12 — UX 패스 2 + 안정화

- 기기 **별명**(목록·캔버스·트레이 일괄 적용), 발견·최근을 합친 **기기 추가 목록**(중복 제거, IP→기계명 승격)
- 연결 코드 **재발급/복사/가리기**, 리치 트레이 메뉴(기기별 끊기) + **상태 LED 아이콘**
- `Ctrl+Alt+←→` **기기 전환**, **배치 프리셋**(집/사무실), 조작 화면 스크롤 + 창 크기 기억
- **투명 커서 미아 수정** — 끊김 시 즉시 복귀 + 실행 시 무조건 복원
- **발견 방화벽 수정** — 업데이트마다 규칙이 옛 exe를 가리키던 문제 (경로 검사 후 재등록, 서브넷 브로드캐스트 추가)

## v0.11 — UX 패스 1

- 역할 선택 원스텝(풀블리드 분할, 수신 선택 즉시 대기), 헤더 **상태 필**(모든 탭에서 연결 상태 확인·호버로 끊기)
- **토스트 알림**(연결/끊김/Ctrl×3 팁), 연결 코드 **6칸 입력**(자동 연결·인증 실패 인라인 표시)
- 배치 탭 **인스펙터 패널**, **경계 통과 시각화**(통과 구간 ‹·›, 범위 밖 ✕), 캔버스 **휠 확대/이동**

## v0.10 — 멀티 수신 허브

- 조작 기기 1대가 여러 수신 기기에 동시 접속. 모두 하나의 평면에 놓이고, 커서/키/클립보드는 각 수신 기기로 라우팅. 연결 화면에서 여러 대 추가·개별 표시

## v0.9 — 통합 · 보안 · 발견 · 파일

- **v0.9.0**: 앱 통합 — 세 팝업 창을 **하나의 창 + 3탭(연결·모니터 배치·설정)** 으로. 레이아웃 자동 저장, 모니터 변경 자동 재감지, 개발용 화면 제거
- **v0.9.3**: **전송 암호화** — ECDH(nistP256) → 방향별 AES-256-GCM, 연결 코드로 핸드셰이크 인증(도청·MITM 방지), 리플레이 방지 카운터. TCP KeepAlive → 자동 재연결. **비상 복귀 핫키**(`Ctrl` 3연타)
- **v0.9.4**: **LAN 자동 발견** — 수신 대기 중 UDP 비컨 방송, 조작 기기에 "같은 네트워크에서 발견됨" 자동 표시. PIN은 방송 안 함
- **v0.9.6**: **파일 클립보드** — 탐색기 Ctrl+C → 상대 PC에서 Ctrl+V (`%LOCALAPPDATA%\Snapfield\Received`, 합계 32MB)

## v0.8 — 클립보드 + 페어링 (초기 계획의 종착점)

- **클립보드 공유(텍스트 + 이미지)**: 양방향, 에코 루프 방지, 텍스트 500KB / 이미지 PNG 8MB
- **연결 코드(PIN)**: 수신 PC가 6자리 코드 표시 → 코드 불일치 시 차단 + 재시도 중단
- v0.8.2~0.8.6: 노트북/모니터 자동 구분(EDID INTERNAL), 촉각적 기기 오브젝트 캔버스, Aurora 다크 톤, 네트워크 화면 재설계(조작/수신 명칭, 최근 연결 원터치)

## v0.7 — 트레이 상주 + 자동화

- 창을 닫아도 백그라운드에서 연결 유지 (트레이 우클릭 → 종료가 실제 종료)
- **로그인 시 자동 실행**, **실행 시 마지막 연결 복원**, 자동 재연결 → 부팅하면 알아서 다시 붙음
- Listen 시 **방화벽 규칙 자동 등록** (UAC 1회)

## v0.6 — 보정 UI 기반 크로스머신 배치 (핵심 차별점)

- 연결하면 상대 PC 모니터가 `layout.json`에 병합되고 보정 캔버스에 주황색으로 표시
- 드래그로 실제 책상 배치대로 놓으면 → 그 배치대로 커서가 넘어감 (저장 즉시 라이브 반영)
- 모니터 사이 틈은 30mm까지 자동 브리지 (엣지 프로브 2단)
