# Snapfield 프로그램 소개

여러 PC와 안드로이드·리눅스, 하나의 책상으로 — v0.16.8 기준

**Snapfield**는 여러 Windows PC와 안드로이드 폰·태블릿, 리눅스 데스크톱의 화면을 하나의 평면에 놓고, 마우스·키보드·클립보드를 화면 경계 너머로 공유하는 입력 공유 도구다. 조작하는 Windows PC 1대(**조작 기기**)와 조작을 받는 PC/폰/리눅스 여러 대(**수신 기기**)로 구성되며, 외부 서버 없이 같은 로컬 네트워크 안에서만 동작한다.

> **다운로드**: [GitHub 릴리스](https://github.com/Q07K/Snapfield/releases) — `Snapfield-vX.Y.Z.exe`(PC), `Snapfield-vX.Y.Z.apk`(안드로이드), `Snapfield-Receiver-vX.Y.Z-linux-{x64,arm64}`(리눅스). 모두 설치 과정 없는 단일 파일.
> **설치 방법**: [설치 가이드](install-guide.md) 참고.

## 1. 왜 만들었나

책상 위 기기가 늘어날수록 작업 흐름이 끊긴다:

- **기기마다 입력 장치 한 세트** — PC 두 대에 폰까지, 기기를 바꿀 때마다 손이 키보드와 마우스를 옮겨 다닌다.
- **픽셀 기준 도구의 어긋난 경계** — 기존 공유 도구는 픽셀 좌표로 화면을 잇는다. 해상도·배율이 다른 모니터가 섞이면 경계에서 커서가 엉뚱한 높이로 튄다.
- **폰과 태블릿은 언제나 예외** — 스크린샷 하나 PC로 옮기는 데도 메신저에 "나에게 보내기".

## 2. 핵심 아이디어 — 전역 물리 평면

Snapfield는 화면을 픽셀이 아니라 **실제 물리 크기(mm)** 로 다룬다.

- 모든 화면을 실제 크기 그대로 하나의 mm 좌표 평면에 배치한다.
- 커서 이동은 `픽셀 → mm 평면 → 상대 기기 픽셀` 세 단계 좌표 변환으로 계산된다.
- 그 결과 4K와 FHD, 27인치 모니터와 6인치 폰이 섞여도 **경계에서 커서 위치가 실제 높이 그대로 이어진다**.
- 책상 위에서 떨어져 있는 기기 사이의 간격도 그대로 건너뛴다.

## 3. 주요 기능

| 기능 | 설명 |
|---|---|
| 커서·키보드 공유 | 커서를 화면 끝으로 밀면 인접 기기로 넘어간다. `Ctrl+Alt+←/→`로 기기 전환 스트립(선택 후 이동, 도착 지점 펄스), `Ctrl` 3연타로 비상 복귀 |
| 클립보드 동기화 | 텍스트·이미지 양방향 공유. PC 사이에는 파일 복사까지 |
| 다대다 허브 | 조작 PC 한 대가 여러 수신 기기를 동시에 연결·조작 |
| 안드로이드 수신 | 폰·태블릿이 평면 위의 모니터가 된다 — 탭·드래그·스크롤·한글 입력 |
| 리눅스 수신 | 우분투 데스크톱(Wayland/X11, x64·arm64)이 평면 위의 모니터가 된다 — 커널 레벨 uinput 주입, 클립보드 양방향 |
| 배치 캔버스 | 실제 책상 배치 그대로 드래그. 놓는 즉시 저장·실시간 반영, 휠 줌, 프리셋 |
| 종단 간 암호화 | ECDH 키 교환 + AES-256-GCM, 6자리 연결 코드 인증 |

### 모니터 배치 캔버스

- 기기를 드래그해 놓는 순간 저장되고 연결에 실시간 반영된다.
- **조용한 캔버스** — 커서가 통과할 수 있는 경계 화살표는 드래그 중에만 초록으로 표시된다.
- 마우스 휠 줌·팬으로 작은 폰 화면도 정밀하게 배치할 수 있다.
- 배치 프리셋(사무실/집 등)을 저장해 두고 한 번에 전환한다.
- 기기 종류(모니터·노트북·태블릿·폰)를 구분해 표시한다.

### 안드로이드 수신

안드로이드 기기는 접근성 서비스와 전용 키보드(IME)로 평면 위의 모니터가 된다:

- **접근성 커서** — PC 마우스로 탭·드래그·스크롤. 화면 위에 커서가 그려진다.
- **Snapfield 키보드** — PC 키보드 그대로 입력. 한/영 전환과 한글 두벌식 조합 지원.
- **PrtSc 캡처** — PC에서 `PrtSc` 한 번으로 폰 화면이 PC 클립보드에 들어온다.
- **키 매핑** — `Esc`=뒤로, `Win`=홈, `Alt+Tab`=최근 앱, 우클릭=뒤로, 휠클릭=최근 앱, 볼륨·미디어 키.
- **화면 유지** — 조작 중에는 폰 화면이 꺼지지 않게 유지한다.

전체 키 매핑과 설정 절차는 [설치 가이드](install-guide.md#5-단축키--키-매핑)에 정리되어 있다.

### 리눅스 수신

우분투 데스크톱은 헤드리스 콘솔 데몬으로 평면 위의 모니터가 된다 (x64·arm64):

- **커널 레벨 입력** — `/dev/uinput` 가상 마우스·키보드로 주입하므로 Wayland/X11 어느 세션이든 진짜 시스템 커서가 움직인다.
- **한글 입력** — 키가 물리 위치 그대로 전달되어 한/영 키가 리눅스 ibus 입력기로 그대로 토글된다.
- **클립보드 양방향** — 텍스트·이미지. 세션에 맞춰 wl-clipboard(Wayland) 또는 xclip(X11)을 사용하며, 최초 실행 시 자동 설치를 제안한다.

## 4. 보안

모든 전송은 종단 간 암호화된다. 연결 절차:

1. **6자리 연결 코드** — 수신 기기 화면에만 표시된다. 코드 원문은 네트워크로 전송하지 않고 HMAC으로 검증한다.
2. **ECDH 키 교환** — P-256 타원곡선으로 세션마다 새 암호화 키를 협상한다.
3. **AES-256-GCM** — 모든 입력·클립보드 데이터를 인증 암호화한다. 재전송 공격 방지 카운터 포함.

추가로:

- 로컬 네트워크 전용 — 외부 서버·클라우드를 거치지 않는다.
- 연결 코드는 언제든 재발급할 수 있고, 기기별 연결 해제를 지원한다.
- 데스크톱(C#)과 안드로이드(Kotlin)가 동일한 프로토콜을 바이트 단위로 호환되게 구현한다.

## 5. 아키텍처

순수 로직과 플랫폼 코드를 분리한 모노레포:

```
Snapfield/
├─ src/
│  ├─ Snapfield.Core       좌표 변환 · 경계 탐지 · 커서 라우팅 (순수 로직)
│  ├─ Snapfield.Platform   입력 후킹 · 네트워크 · 클립보드 (Win32)
│  └─ Snapfield.App        WPF UI · 배치 캔버스 · 트레이 (단일 exe)
├─ android/                Kotlin 수신 앱 (접근성 커서 · IME · 수신 서비스)
├─ tests/                  Core 단위 테스트 56개
└─ docs/                   설치 가이드 · 소개 문서
```

- **Core**는 Windows 의존성이 없는 순수 C# — 좌표·경계·라우팅 로직 전체가 단위 테스트로 덮여 있다.
- **android/** 는 데스크톱과 같은 와이어 프로토콜(핸드셰이크·암호화·메시지 인코딩)을 Kotlin으로 구현한다. 고빈도 입력 메시지(커서·버튼·휠·키)는 바이너리 fast-path로, 나머지는 JSON으로 전송된다.
- **CI/릴리스** — 버전 태그(`v*`)를 푸시하면 GitHub Actions가 exe와 서명된 APK를 자동 빌드해 릴리스에 첨부한다.

## 6. 시작하기

1. **설치** — 릴리스에서 exe(PC)/APK(폰)를 받는다. 설치 과정 없는 단일 파일. PC에서는 실행 시 관리자 승인(UAC)이 필요하다 — 관리자 권한 창 위에서도 원격 조작이 동작하기 위함.
2. **연결** — 수신 기기 화면의 초록 6자리 코드를 조작 PC에 입력한다. 다음부터는 원탭 재연결.
3. **배치** — 배치 캔버스에서 기기를 실제 책상 위치대로 드래그한다. 커서를 경계로 밀면 넘어간다.

자세한 절차(SmartScreen·Play 프로텍트·접근성 권한 등)는 [설치 가이드](install-guide.md)에.
